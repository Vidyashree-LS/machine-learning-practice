Scaling 
* It is a preprocessing technique which scales the values equally for all the columns to give equal importance to all the columns.

when to use Scaling?
* only for numerical column not for object column.
* when numerical column values are in different range then we use scaling technique.
* we use this only on feature columns.

Why it is important?
* To give equal importanceto all the columns.

Not to use scaling
* for tree based models scaling techniques are not required
* when the feature columns have same range values

where we use scaling techniques?
* Linear Regression
* Logistic Regression
* KNN
* SVM
* K Means clustering
* PCA
* Regularization


Standardization
* It is a type of scaling technique.
* it will not the scales the values not between fixed range
* we use this when we have outliers

Normalization
* It is also a type of scaling technique
* It will scales the values but between the fixed range
* The range is between 0 and 1
* when we dont have outliers and the data is clean we use Normalization

Standard scaler
* It is a scaling preprocessing technique which sclaes the values between no range
* we use this when we have less outliers
* internally it uses z-score formula and scales the values useing its formula
* after scaling we get mean near to 0 and standard deviation near to 1
* Formula - (x-mean)/standard deviation

Robust scaler
* it is also a scalingpreprocessing technique which sclaes the vslues between no range
* we use this when we have more outliers (because it uses median)
* Formula - (x-Q2)/IQR

MinMaxScaler
* Formula - (x-xmin)/(xmax-xmin)

LabelEncoder
what is Label Encoder?
* It is encoding preprocessing technique which accepte one column and encodes the values from zero

Syntax:
from sklearn.preprocessing import LabelEncoder
encoder = LabelEncoder()
encoder.fit_transform(xtrain[colname])
encoder.fit_transform(xtrain[[colname]])
encoder.transform(xtest[colname])
encoder.transform(xtest[[colname]])

How it works?
It accepts only one column ----> Selects the unique values ----> Sorts by alphabetical(ascending) order ----> assigns the values from zero
  col 1   
    a11             selects unique values             Sorts by alphabetical order              assigns the values from zero
    a12    ---->     a11, a12,b1 ,a13      ----->      a11, a12, a13, b1           ------>       a11  -   0  
    b1                                                                                           a12  -   1
    a13                                                                                          a13  -   2
    b1                                                                                           b1   -   3


When to use label encoder?
* when we have only one categorical column, then we use label encoder
* For less unique values we use label encoder (<=10)
* Mostly we use this for target column (classification scenarios)

When not to use label encoder?
* when we have more categorical columns then we should not use label encoder
* when we have more unique values then we should not use label encoder (>10)

* fit_transform -> it findss the labels and then assigns the vlaues
* transform ->

OneHot Encoder
What is onehot encoder?
* It is encoding preprocessing technique which accepts 2D and encodes binary values

Syntax:
from sklearn.preprocessing import OneHot Encoder
encoder = OneHot Encoder(handle_unknown="ignore"|"error"(default), spare_output = True(default) | False)

when to use OneHot Encoder?
* when feature columns are having two or more columns then we use onehot encoder
* For less unique values we use label encoder (<=10)

when not to use OneHot Encoder?
* when we have more unique vlaues
